# 普通回溯

## [电话号码的字母组合](https://leetcode.cn/problems/letter-combinations-of-a-phone-number/)(17)

问题：给定一个仅包含数字 `2-9` 的字符串，返回所有它能表示的字母组合。答案可以按 **任意顺序** 返回。给出数字到字母的映射如下（与电话按键相同）。注意 1 不对应任何字母。

分析：

* 当前操作：枚举 `path[i]` 填入字符
* 子问题：构造字符串 `>= i` 的部分
* 下一个子问题：构造字符串 `>= i + 1` 的部分

复杂度：`O(n4`^n^`)` `O(n)` 其中 `n` 为 `digits` 的长度。最坏情况下每次需要枚举 `4` 个字母，递归次数为一个满四叉树的节点个数，那么一共会递归 `O(4`^n^`)` 次（等比数列和），再算上加入答案时需要 `O(n)` 的时间，所以时间复杂度为 `O(n4`^n^`)`。

```java
private static final String[] MAPPING = new String[]{"", "", "abc", "def", "ghi", "jkl", "mno", "pqrs", "tuv", "wxyz"};
private static final List<String> ans = new ArrayList<>();
private char[] digits, path;  // 长度固定则用path数组

private void dfs(int i) {  // 含义是字符串下标>=i的子问题
    if (i == digits.length) {
        ans.add(new String(path));
        return;
    }
    for (char c : MAPPING[digits[i] - '0'].toCharArray()) {
        path[i] = c;  // 当前操作：枚举path[i]填入字符
        dfs(i + 1);  // 下一个子问题，构造字符串>=i+1的部分
    }
}

public List<String> letterCombinations(String digits) {  // 这题是多个选一个，是必选，不能不选
    int n = digits.length();
    if (n == 0) {
        return ans;
    }
    this.digits = digits.toCharArray();
    path = new char[n];
    dfs(0);
    return ans;
}
```

# 子集型回溯

## [子集](https://leetcode.cn/problems/subsets/)(78)

问题：给你一个整数数组 `nums` ，数组中的元素 **互不相同** 。返回该数组所有可能的**子集（幂集）**。解集 **不能** 包含重复的子集。你可以按 **任意顺序** 返回解集。

解法1：子集型回溯（输入视角）：每个元素可以选/不选

* 当前操作：枚举第 `i` 个数选或不选
* 子问题：从下标 `>= i` 的数字中构造子集
* 下一个子问题：从下标 `>= i + 1` 的数字中构造子集

复杂度：`O(n2`^n^`)` `O(n)` 其中 `n` 为 `nums` 的长度。每次都是选或不选，递归次数为一个满二叉树的节点个数，那么一共会递归 `O(2`^n^`)` 次（等比数列和），再算上加入答案时复制 `path` 需要 `O(n)` 的时间，所以时间复杂度为 `O(n2`^n^`)`。

```java
private final List<List<Integer>> ans = new ArrayList<>();
private final List<Integer> path = new ArrayList<>();
private int[] nums;

public List<List<Integer>> subsets(int[] nums) {
    this.nums = nums;
    dfs(0);
    return ans;
}

private void dfs(int i) {  // 子问题：从下标 >=i 的数字中构造子集
    if (i == nums.length) {
        ans.add(new ArrayList<>(path));  // 固定答案
        return;
    }
    dfs(i + 1);  // 不选 nums[i]
    path.add(nums[i]);
    dfs(i + 1);  // 选 nums[i]
    path.remove(path.size() - 1);  // 恢复现场
}
```

解法2：子集型回溯（答案视角）：每次必须选一个数，关键是选哪个数

* 当前操作：枚举一个下标 `j > i` 的数字，加入 `path`
  * 由于不重复，即不考虑顺序，那么就规定一个严格递增的顺序，每次枚举必须有 `j > i`
* 子问题：从下标 `>= i` 的数字中构造子集
* 下一个子问题：从下标 `>= i + 1` 的数字中构造子集

复杂度：`O(n2`^n^`)` `O(n)` 其中 `n` 为 `nums` 的长度。答案的长度为子集的个数，即 `2`^n^ ，同时每次递归都把一个数组放入答案，因此会递归 `2`^n^ 次，再算上加入答案时复制 `path` 需要 `O(n)` 的时间，所以时间复杂度为 `O(n2`^n^`)`。

```java
private void dfs2(int i) {  // 子问题：从下标 >=i 的数字中构造子集
    ans.add(new ArrayList<>(path));  // 递归到的每个节点都是答案
    // if (i == nums.length) { return; }  // for循环保证了i递归到 nums.length 时无代码可执行，自动返回
    for (int j = i; j < nums.length; j++) {  // 不考虑顺序，就规定一个顺序j>=i，枚举选择的数字
        path.add(nums[j]);
        dfs2(j + 1);  // 下一个子问题：从下标>=j+1的数字中构造子集
        path.remove(path.size() - 1);  // 恢复现场
    }
}
```

## [分割回文串](https://leetcode.cn/problems/palindrome-partitioning/)(131)

问题：给你一个字符串 `s`，请你将 `s` 分割成一些子串，使每个子串都是 **回文串** 。返回 `s` 所有可能的分割方案。

分析：转化为**枚举逗号的位置（子集型回溯）**，截取逗号间子串并判断回文，**注意最后一个逗号一定要取到（保证回文才能加入答案）**

解法1（输入视角）：逗号选或不选

* 当前操作：枚举第 `i` 个逗号选或不选
* 子问题：从下标 `>= i` 的后缀中构造回文分割
* 下一个子问题：从下标 `>= i + 1` 的后缀中构造回文分割

复杂度：`O(n2`^n^`)` `O(n)` 其中 `n` 为 `s` 的长度。每次都是选或不选，递归次数为一个满二叉树的节点个数，那么一共会递归 `O(2`^n^`)` 次（等比数列和），再算上判断回文和加入答案时需要 `O(n)` 的时间，所以时间复杂度为 `O(n2`^n^`)`。

```java
private final List<List<String>> ans = new ArrayList<>();
private final List<String> path = new ArrayList<>();
private String s;

private boolean isPalindrome(int left, int right) { // 相向双指针判断是否回文
    while (left < right) {
        if (s.charAt(left++) != s.charAt(right--)) {
            return false;
        }
    }
    return true;
}

public List<List<String>> partition(String s) {
    this.s = s;
    dfs(0, 0);
    return ans;
}
// 因为要截取子串，所以只枚举第i个逗号不够，还需将起始位置start作为递归参数
private void dfs(int i, int start) {  // start 表示当前这段回文子串的开始位置
    if (i == s.length()) {
        ans.add(new ArrayList<>(path));  // 固定答案
        return;
    }
    if (i < s.length() - 1) {  // i == n-1 时右边没有逗号，或理解为最右端的虚拟逗号必选，相当于一个终止符
        dfs(i + 1, start);  // 不选 i 和 i+1 之间的逗号
    }
    if (isPalindrome(start, i)) {  // 若不是回文子串则只能不选，继续递归
        path.add(s.substring(start, i + 1));  // 实际上是截取了 [start, i+1) == [start, i] 的子串
        dfs(i + 1, i + 1);  // 选 i 和 i+1 之间的逗号，下一个子串的位置从 i+1 开始
        path.remove(path.size() - 1);  // 恢复现场
    }
}
```

解法2**（答案视角）：枚举子串结束位置**

* 当前操作：枚举一个 `j > i` 的回文子串 `s[i...j]` 加入 `path`
* 子问题：从下标 `>= i` 的后缀中构造回文分割
* 下一个子问题：从下标 `>= j + 1` 的后缀中构造回文分割

复杂度：`O(n2`^n^`)` `O(n)` 其中 `n` 为 `s` 的长度。答案的长度至多为逗号子集的个数，即 `O(2`^n^`)` ，因此会递归 `O(2`^n^`)` 次，再算上判断回文和加入答案时需要 `O(n)` 的时间，所以时间复杂度为 `O(n2`^n^`)`。

```java
private void dfs2(int i) {  // i 表示当前这段回文子串的开始位置
    if (i == s.length()) {
        ans.add(new ArrayList<>(path));  // 固定答案
        //  return;
    }
    for (int j = i; j < s.length(); j++) {  // 枚举 j 作为子串的结束部分
        if (isPalindrome(i, j)) {  // 若不是回文子串则只能不选，继续递归
            path.add(s.substring(i, j + 1));  // 实际上是截取了 [i, j+1) == [i, j] 的子串
            dfs2(j + 1);
            path.remove(path.size() - 1);  // 恢复现场
        }
    }
}
```

# 组合型回溯

## [组合](https://leetcode.cn/problems/combinations/)(77)

问题：给定两个整数 `n` 和 `k`，返回范围 `[1, n]` 中所有可能的 `k` 个数的组合。你可以按 **任何顺序** 返回答案。

解法1：输入视角：选或不选

复杂度：`O(k*C(n,k))` `O(k)`

```java
private final List<List<Integer>> ans = new ArrayList<>();
private final List<Integer> path = new ArrayList<>();
private int k;

public List<List<Integer>> combine(int n, int k) {
    this.k = k;
    dfs(n);  // 从大到小枚举
    return ans;
}

private void dfs(int i) {  // 表示接下来可在 [1...i] 的范围内枚举
    int d = k - path.size();  // 还要选 d 个数
    if (d == 0) {
        ans.add(new ArrayList<>(path));
        return;
    }
    if (i > d) {  // i == d 时 [1...i] 必须全选，否则长度将达不到k
        dfs(i - 1);
    }
    path.add(i);
    dfs(i - 1);
    path.remove(path.size() - 1);  // 恢复现场
}
```

解法2：答案视角：枚举下一个数选哪个

复杂度：`O(k*C(n,k))` `O(k)`

```java
public void dfs2(int i) {  // 表示接下来可在 [1...j] 的范围内枚举
    int d = k - path.size();  // 还要选 d 个数
    if (d == 0) {
        ans.add(new ArrayList<>(path));
        return;
    }
    //  if (i < d) { return; }  // 剪枝，合并到for循环中
    for (int j = i; j >= d; j--) {  // j == d 时 [1...j] 必须全选，否则长度将达不到k
        path.add(j);
        dfs2(j - 1);
        path.remove(path.size() - 1);  // 恢复现场
    }
}
```

## [组合总和 III](https://leetcode.cn/problems/combination-sum-iii/)(216)

问题：找出所有相加之和为 `n` 的 `k` 个数的组合，且满足下列条件：

- 只使用数字1到9
- 每个数字 **最多使用一次** 

返回 *所有可能的有效组合的列表* 。该列表不能包含相同的组合两次，组合可以以任何顺序返回。

解法1：输入视角：选或不选

复杂度：`O(k*C(9,k))` `O(k)`

```java
private final List<List<Integer>> ans = new ArrayList<>();
private final List<Integer> path = new ArrayList<>();
private int k;

public List<List<Integer>> combinationSum3(int k, int n) {  // 这里的 n 含义是target
    this.k = k;
    dfs(9, n);
    return ans;
}
// 为了剪枝，增加参数t表示接下来要选的d个数的目标和（若不增加参数t，仅在加入答案时判断和是否为n，太臃肿）
private void dfs(int i, int t) {  // 表示接下来可在 [1...i] 的范围内枚举
    int d = k - path.size();  // 还要选 d 个数
    if (t < 0 || t > (i * 2 - d + 1) * d / 2) // 剪枝，剩余的target过小或过大导致不满足
        return;
    if (d == 0) {
        ans.add(new ArrayList<>(path));  // 路径加入答案时必然有 t == 0 ，无需判断
        return;
    }
    if (i > d) {  // i == d 时 [1...i] 必须全选，否则长度将达不到k
        dfs(i - 1, t);
    }
    path.add(i);
    dfs(i - 1, t - i);
    path.remove(path.size() - 1);  // 恢复现场
}
```

解法2：答案视角：枚举下一个数选哪个

复杂度：`O(k*C(9,k))` `O(k)`

    }
}

## [组合总和](https://leetcode.cn/problems/combination-sum/)(39)

问题：给你一个 **无重复元素** 的整数数组 `candidates` 和一个目标整数 `target` ，找出 `candidates` 中可以使数字和为目标数 `target` 的 所有 **不同组合** ，并以列表形式返回。你可以按 **任意顺序** 返回这些组合。`candidates` 中的 **同一个** 数字可以 **无限制重复被选取** 。如果至少一个数字的被选数量不同，则两种组合是不同的。 对于给定的输入，保证和为 `target` 的不同组合数少于 `150` 个。

解法1：输入视角：选或不选

```java
private final List<List<Integer>> ans = new ArrayList<>();
private final List<Integer> path = new ArrayList<>();
private int[] candidates;
private int n;

public List<List<Integer>> combinationSum(int[] candidates, int target) {
    this.candidates = candidates;
    n = candidates.length;
    Arrays.sort(this.candidates);  // 先排序再剪枝
    dfs(0, target);
    return ans;
}

private void dfs(int i, int target) {
    // 不会出现 i 和 target 同时到达边界值的情况（因为只能由不选递归得到，且一定无解），因此两个 if 的顺序不重要
    if (target == 0) {
        ans.add(new ArrayList<>(path));
        return;
    }
    if (i == n) { // i == n 表明已遍历完 candidates[i] 的所有元素，因为不存在 candidates[n]
        return;
    }
//    if (candidates[i] > target) { // 剪枝（也可以放在这里）
//        return;
//    }
    dfs(i + 1, target);  // 不选
    if (target - candidates[i] >= 0) {  // 剪枝，同时避免死循环
        path.add(candidates[i]);
        dfs(i, target - candidates[i]);  // 选，注意 i 不变，表示在下次递归中可以继续选 candidates[i]
        path.remove(path.size() - 1);
    }
}
```

解法2：答案视角：枚举选哪个

```java
private void dfs2(int i, int target) {
    if (target == 0) {
        ans.add(new ArrayList<>(path));
        return;
    }
//    if (candidates[i] > target) { // 剪枝（也可以放在这里）
//        return;
//    }
    for (int j = i; j < n; j++) { // 保证了递归参数 i 不会到 n（后面 dfs 也只会递归到 j）
        if (target - candidates[j] < 0) {  // 剪枝，同时避免死循环
            break;  // return;
        }
        path.add(candidates[j]);
        dfs2(j, target - candidates[j]); // 注意这里是递归到 j 不是 j+1，表示 candidates[j] 可以重复选取。
        path.remove(path.size() - 1);
    }
}
```

## [组合总和 II](https://leetcode.cn/problems/combination-sum-ii/)(40)

问题：给定一个候选人编号的集合 `candidates` 和一个目标数 `target` ，找出 `candidates` 中所有可以使数字和为 `target` 的组合。`candidates` 中的每个数字在每个组合中只能使用 **一次** 。**注意：**解集不能包含重复的组合。

解法1：输入视角：选或不选

```java
private List<List<Integer>> ans = new ArrayList<>();
private final List<Integer> path = new ArrayList<>();
private int[] candidates;
private int n;

public List<List<Integer>> combinationSum2(int[] candidates, int target) {
    this.candidates = candidates;
    n = candidates.length;
    Arrays.sort(this.candidates);  // 先排序再剪枝
    dfs(0, target);
    ans = new ArrayList<>(new HashSet(ans));  // 选或不选时需要最终对答案去重，可能选到不同位置的重复数字
    return ans;
}

private void dfs(int i, int target) {
	// 可能出现i和target同时到达边界值的情况（因为可以由选或不选递归得到），必须先判断target，否则选择最后一个数时会被漏掉
    if (target == 0) {
        ans.add(new ArrayList<>(path));
        return;
    }
    if (i == n) { // i == n 表明已遍历完 candidates[i] 的所有元素，因为不存在 candidates[n]
        return;
    }
//    if (candidates[i] > target || (n - i) * candidates[n - 1] < target) { // 剪枝（也可以放在这里）
//        return;
//    }
    dfs(i + 1, target);  // 不选
    if (target - candidates[i] >= 0 || target - (n - i) * candidates[n - 1] <= 0) {  // 剪枝，同时避免死循环
        path.add(candidates[i]);
        dfs(i + 1, target - candidates[i]);  // 选
        path.remove(path.size() - 1);
    }
}
```

解法2：答案视角：枚举选哪个

```java
private void dfs2(int i, int target) {
    if (target == 0) {
        ans.add(new ArrayList<>(path));
        return;
    }
//    if (candidates[i] > target || (n - i) * candidates[n - 1] < target) {
//        return; // 剪枝放在这里 candidates[i] 的 i 可以取到 n，导致越界
//    }
    for (int j = i; j < n; j++) { // j == n - 1 时，dfs2(j + 1) 会递归到 n，所以剪枝放在外面会越界
        if (j > i && candidates[j] == candidates[j - 1]) {  // 去掉重复项，只取第一个数进行递归
            continue; // 保证了搜索时不重复，比得到答案再去重效率更高
        }
        if (candidates[j] > target || (n - j) * candidates[n - 1] < target) {  // 剪枝，第一个最重要
            break;  // return;
        }
        path.add(candidates[j]);
        dfs2(j + 1, target - candidates[j]);
        path.remove(path.size() - 1);
    }
}
```

## [括号生成](https://leetcode.cn/problems/generate-parentheses/)(22)

问题：数字 `n` 代表生成括号的对数，请你设计一个函数，用于能够生成所有可能的并且 **有效的** 括号组合。

分析：组合型回溯：`C(2n,n)`，**转化为从 `2n` 个位置中选 `n` 个位置放左括号，左括号等价于选，右括号等价于不选**

解法1：**输入视角：枚举选左括号还是右括号**

* 当前操作：枚举 `path[i]` 是左括号还是右括号（选或不选）
* 子问题：构造字符串 `>= i` 的部分
* 下一个子问题：构造字符串 `>= i + 1` 的部分

复杂度：`O(n*C(2n,n))` `O(n)` 但由于左右括号的约束，实际上没有这么多叶子，根据 `Catalan` 数，只有 `C(2n,n)/(n+1)`，所以实际的时间复杂度为`O(C(2n,n))`。此外，根据阶乘的 `Stirling` 公式，时间复杂度也可以表示为 `O(4`^n^`/n`^1/2^`)`

```java
private final List<String> ans = new ArrayList<>();
private final List<Integer> path2 = new ArrayList<>();  // 动态维护左括号的下标
private char[] path;  // 长度固定则用path数组
private int n;

public List<String> generateParenthesis(int n) {
    this.n = n;
    path = new char[n * 2];
    dfs(0, 0);
    return ans;
}
// 因为能否枚举左括号和右括号会遭到限制，不是简单的任意挑选组合C(n,k)，故引入左括号的数量open，在递归中描述这种限制
private void dfs(int i, int open) {  // 含义为前i个字符中左括号的数量为open
    if (i == 2 * n) {
        ans.add(new String(path));
    }
    if (open < n) {  // 可以填左括号
        path[i] = '(';
        dfs(i + 1, open + 1);
    }
    if (i - open < open) {  // 可以填右括号
        path[i] = ')';
        dfs(i + 1, open);
    }
}
```

解法2：答案视角：枚举下一个左括号的位置

复杂度：`O(n*C(2n,n))` `O(n)` 但由于左右括号的约束，实际上没有这么多叶子，根据 `Catalan` 数，只有 `C(2n,n)/(n+1)`，所以实际的时间复杂度为`O(C(2n,n))`。此外，根据阶乘的 `Stirling` 公式，时间复杂度也可以表示为 `O(4`^n^`/n`^1/2^`)`

```java
private void dfs2(int i, int balance) {  // 前i个字符已填，balance = 左括号个数 - 右括号个数
    if (path2.size() == n) { // path2 记录枚举左括号的位置下标
        char[] path = new char[n * 2];
        Arrays.fill(path, ')');
        for (int j : path2) {
            path[j] = '(';
        }
        ans.add(new String(path));
        return;
    }
    for (int close = 0; close <= balance; close++) { // 可以填 0 到 balance 个右括号，选择连续填 close 个右括号
        path2.add(i + close); // 填完 close 个右括号后，填 1 个左括号，继续递归下去
        dfs2(i + close + 1, balance - close + 1);
        path2.remove(path2.size() - 1);  // 恢复现场
    }
}
```

# 排列型回溯

## [全排列](https://leetcode.cn/problems/permutations/)(46)

问题：给定一个**不含重复数字**的数组 `nums` ，返回其 *所有可能的全排列* 。你可以 **按任意顺序** 返回答案。

分析：排列型回溯：`A(n,n)`，数组 `path` 记录路径上已选的数，集合 `s` 记录剩余未选数字

* 当前操作：从 `s` 中枚举 `path[i]` 要填的数字 `x`
* 子问题：构造排列 `>= i` 的部分，剩余未选数字集合为 `s`
* 下一个子问题：构造排列 `>= i + 1` 的部分，剩余未选数字集合为 `s - {x}`

复杂度：`O(n*n!)` `O(n)` 其中 `n` 为 `nums` 的长度。根据等比数列求和估计，搜索树中的节点个数低于 `3⋅n!`。实际上，精确值为 `A(n,0) + ... + A(n,n-1) + A(n,n) = ⌊e⋅n!⌋ = O(n!)`。此外，每个非叶节点要花费 `O(n)` 的时间遍历 `onPath` 数组，每个叶结点也要花费 `O(n)` 的时间复制 `path` 数组，因此时间复杂度为 `O(n*n!)`。

```java
private final List<List<Integer>> ans = new ArrayList<>();
private final List<Integer> path = new ArrayList<>();
private boolean[] onPath;
private int[] nums;

private void dfs(int i) {
    if (i == nums.length) {
        ans.add(new ArrayList<>(path));
        return;
    }
    for (int j = 0; j < nums.length; j++) {  // 因为是排列型不是组合型，因此 j 从 0 开始而不是从 i 开始枚举
        if (!onPath[j]) {
            onPath[j] = true;  // 打上标记
            path.add(nums[j]);
            dfs(i + 1);
            onPath[j] = false;  // 恢复现场
            path.remove(path.size() - 1);
        }
    }
}

public List<List<Integer>> permute(int[] nums) {
    this.nums = nums;
    onPath = new boolean[nums.length];  // Java里系统默认赋初值 false
    dfs(0);
    return ans;
}
```

## [全排列 II](https://leetcode.cn/problems/permutations-ii/)(47)

问题：给定一个可**包含重复数字**的序列 `nums` ，***按任意顺序*** 返回所有不重复的全排列。

核心思想：所有相同的数字只能有一种访问顺序！即 `x1,x2,...,xn` **`!vis[i - 1]` 来去重主要是通过限制一下两个相邻的重复数字的访问顺序**，如对于两个相同的数 `x1, x2`，不做去重的话，会有两种重复排列 `x1x2, x2x1`，而我们只需要取其中任意一种排列。为了达到这个目的，限制一下 `x1, x2` 访问顺序即可，即按顺序只取 `x1x2`。

方法：**遇到相同数字时，只有当 `visit nums[i-1]` 之后才去 `visit nums[i]`**，也就是如果 `!visited[i - 1]` 的话则 `continue`

注意：由于已经提前排序完毕，故所有相同的数字必相邻聚集，因此**对于若干连续的相同数字，只需规定访问 `i - 1` 后再访问 `i` 即可**

复杂度：`O(n*n!)` `O(n)`

```java
private final List<List<Integer>> ans = new ArrayList<>();
private final List<Integer> path = new ArrayList<>();
private boolean[] onPath;
private int[] nums;

private void dfs(int i) {
    if (i == nums.length) {
        ans.add(new ArrayList<>(path));
        return;
    }
    for (int j = 0; j < nums.length; j++) {  // 因为是排列型不是组合型，因此 j 从 0 开始而不是从 i 开始枚举
        if (!onPath[j] && !(j > 0 && nums[j] == nums[j - 1] && !onPath[j - 1])) {  // 去掉重复项
            onPath[j] = true;  // 打上标记
            path.add(nums[j]);
            dfs(i + 1);
            onPath[j] = false;  // 恢复现场
            path.remove(path.size() - 1);
        }
    }
}

public List<List<Integer>> permuteUnique(int[] nums) {
    this.nums = nums;
    Arrays.sort(this.nums);  // 先排序（为了后续去重）
    onPath = new boolean[nums.length];  // Java里系统默认赋初值 false
    dfs(0);
    return ans;
}
```

## [N 皇后](https://leetcode.cn/problems/n-queens/)(51)

问题：按照国际象棋的规则，皇后可以攻击与之处在同一行或同一列或同一斜线上的棋子。**n 皇后问题** 研究的是如何将 `n` 个皇后放置在 `n×n` 的棋盘上，并且使皇后彼此之间不能相互攻击。给你一个整数 `n` ，返回所有不同的 **n 皇后问题** 的解决方案。每一种解法包含一个不同的 **n 皇后问题** 的棋子放置方案，该方案中 `'Q'` 和 `'.'` 分别代表了皇后和空位。

分析：排列型回溯：`A(n,n)`，**设第 `r` 行的皇后在第 `col[r]` 列，则 `col` 是 `[0...n-1]` 的全排列**（保证任意皇后不同行不同列），又因为也不能在同一斜线，利用 **/ 方向 r + c 是定值； \ 方向 r - c 是定值**的性质，放置前判断即可，`n` 皇后需要 `O(n)` 的时间

优化：利用数组或哈希表，将 `r + c` 记录到 `boolean[] diag1` 中，放皇后前判断，若为真则无法放皇后；同理将 `r - c` 记录到 `boolean[] diag2` 中，但为了避免负数需要加上 `n - 1` ，放皇后的时间复杂度由 `O(n)` 降为 `O(1)`

复杂度：`O(n`^2^`*n!)` `O(n)` 搜索树中至多有 `O(n!)` 个叶子，每个叶子生成答案每次需要 `O(n`^2^`)` 的时间，所以时间复杂度为 `O(n`^2^`*n!)`，优化后若不记录答案为`O(3*n!)`。实际上搜索树中远没有这么多叶子，`n=9` 时只有 `352` 种放置方案，远远小于 `9!=362880`。

```java
private final List<List<String>> ans = new ArrayList<>();
private boolean[] onPath, diag1, diag2;  // 均被系统自动初始化为 false
private int[] col;  // 设第 r 行的皇后在第 col[r] 列
private int n;

private void dfs(int r) {
    if (r == n) { // 记录答案 O(n^2)
        List<String> board = new ArrayList<>(n);
        for (int c : col) {
            char[] row = new char[n];
            Arrays.fill(row, '.');
            row[c] = 'Q';
            board.add(new String(row));
        }
        ans.add(board);
        return;
    }
    for (int c = 0; c < n; c++) { // 路径长度 O(n) 实际上会重复计算节点，最终复杂度没有 O(n)
        int rc = r - c + n - 1;  // 避免负数下标
        if (!onPath[c] && !diag1[r + c] && !diag2[rc]) { // 放皇后 O(n) 优化到 O(1)
            onPath[c] = diag1[r + c] = diag2[rc] = true;  // 打上标记
            col[r] = c;  // 自动覆盖式
            dfs(r + 1);
            onPath[c] = diag1[r + c] = diag2[rc] = false;  // 恢复现场
        }
    }
}

public List<List<String>> solveNQueens(int n) {
    this.n = n;
    col = new int[n];
    onPath = new boolean[n];
    diag1 = new boolean[2 * n - 1];
    diag2 = new boolean[2 * n - 1];
    dfs(0);
    return ans;
}
```

## [N 皇后 II](https://leetcode.cn/problems/n-queens-ii/)(52)

问题：**n 皇后问题** 研究的是如何将 `n` 个皇后放置在 `n × n` 的棋盘上，并且使皇后彼此之间不能相互攻击。给你一个整数 `n` ，返回 **n 皇后问题** 不同的解决方案的数量。

分析：仅需计算答案是数量，递归到边界条件 `r == n` 时统计数量即可

复杂度：`O(n!)` `O(n)` 根据通用公式计算为 `O(n*n!)`，但是可以精确算出全排列搜索树的节点个数为 `A(n,0) + ... + A(n,n-1) + A(n,n) = ⌊e⋅n!⌋ = O(n!)`，一般加入答案还需要 `O(n)` 的时间，但本题仅需求解的数量，故计算答案的时间为 `O(1)`，总时间为 `O(n!)`

```java
private boolean[] onPath, diag1, diag2;  // 均被系统自动初始化为 false
private int[] col;  // 设第 r 行的皇后在第 col[r] 列
private int n, ans;

private void dfs(int r) {
    if (r == n) { // 记录答案 O(1)
        ans++;
        return;
    }
    for (int c = 0; c < n; c++) { // 路径长度 O(n) 实际上会重复计算节点，最终复杂度没有 O(n)
        int rc = r - c + n - 1;  // 避免负数下标
        if (!onPath[c] && !diag1[r + c] && !diag2[rc]) { // 放皇后 O(n) 优化到 O(1)
            onPath[c] = diag1[r + c] = diag2[rc] = true;  // 打上标记
            col[r] = c;  // 自动覆盖式
            dfs(r + 1);
            onPath[c] = diag1[r + c] = diag2[rc] = false;  // 恢复现场
        }
    }
}

public int totalNQueens(int n) {
    this.n = n;
    col = new int[n];
    onPath = new boolean[n];
    diag1 = new boolean[2 * n - 1];
    diag2 = new boolean[2 * n - 1];
    dfs(0);
    return ans;
}
```
